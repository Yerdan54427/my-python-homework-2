# 冯嘉晔-25361024-第二次人工智能编程作业

## 一、项目概述

本项目实现了一个命令行考务系统，核心功能包括：

1. 通过学号查找学生信息
2. 随机点名
3. 随机生成考场座位并导出考场安排文件
4. 按座位信息批量生成准考证文件

项目主程序位于 `my-code.py`，输入数据来自 `人工智能编程语言学生名单.txt`。

## 二、任务拆解与 AI 协作策略

为了避免“一次性让 AI 生成全部代码导致难以调试”，我采用了分阶段协作方式：

### 阶段 1：需求建模与边界定义

- 先让 AI 把需求拆成对象和功能模块：`Student` 类、`ExamSystem` 类、主菜单流程。
- 明确输入输出：
	- 输入：学生名单文本
	- 输出：`考场安排.txt` + `准考证/*.txt`
- 先定义异常场景：空文件、格式错误、学号不合法、人数越界。

### 阶段 2：最小可运行版本

- 先完成可运行主链路：加载名单 -> 生成座位 -> 导出文件。
- 验证程序能够在命令行循环运行，菜单分支可达。

### 阶段 3：工程化完善

- 补充输入校验（如 `validate_student_id`、人数必须为整数）。
- 补充异常处理（`FileNotFoundError`、`ValueError`、`EOFError/KeyboardInterrupt`）。
- 优化函数职责：解析、查找、随机抽样、落盘输出分别封装。

### 阶段 4：结果验收与回归

- 验证文件生成数量与学生人数一致。
- 检查文本编码为 UTF-8，中文输出正常。
- 对关键分支做回归：无效菜单输入、超范围点名、空名单场景。

## 三、核心 Prompt 迭代记录

以下为一次典型迭代，展示如何把“能跑”提升到“符合规范”。

### 初代 Prompt

“帮我写一个 Python 程序，读取学生名单，实现随机排座和生成准考证。”

### 初版代码问题

1. 功能混在一个大函数里，可读性差。
2. 缺少异常处理，名单文件不存在会直接崩溃。
3. 输入校验不足，随机点名人数越界时行为不明确。
4. 对中文文本和文件编码细节考虑不完整。

### 优化后的 Prompt（追问）

“请用面向对象方式实现。要求：
1. 必须拆分为 Student 与 ExamSystem 两个类；
2. 只使用课程已学过的标准库（random、os、datetime）；
3. 必须处理 FileNotFoundError、ValueError；
4. 菜单输入要做合法性校验；
5. 生成考场安排与准考证时使用 UTF-8 编码；
6. 每个函数职责单一，不要把所有逻辑写在 main 里。”

### 迭代结果

- 代码结构从“过程式堆叠”改为“类 + 方法分层”。
- 异常处理覆盖更完整，用户输入错误不再导致程序退出。
- 输出文件格式稳定，能够直接用于验收。

## 八、个人复盘

本次作业最大的收获不是“让 AI 写代码”，而是“通过约束和验证驱动 AI 产出可维护代码”。

后续可继续改进的方向：

1. 增加自动化测试（如对 `parse_student_line` 和 `random_pick_students` 的单元测试）。
2. 将输出格式抽为配置项，减少硬编码。
3. 给异常信息增加“用户提示 + 开发调试信息”双层结构，提升可维护性。
